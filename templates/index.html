How This Works

  • This is merely a homepage for our backend database.
  • Our project is dependent on a database, which is deployed on this flask framework.
  • We've used Github to track progress on our project and make issues for each task.
  • Here is a link to our scrum board.

What does the Backend Process look like?

  • The database stores a user's username (name), encrypted password, and the time it takes the user to escape.
  • SQL and SQLAlchemy are used to keep all of the data persistent... meaning that if something were to go wrong with the server or host, then the data would not be lost.
  • The frontend and backend are connected through the use of Javascript's Fetch API, which allows the frontend to call or fetch a backend Flask endpoint.
  • This information that is sent to the backend is considered a request (ie GET, POST). A Python function interprets the request, typically into a setter or getter against an SQL row and provides a response. Examples are user “signs up”, user “login” or "time of completion" updates.
  • Our API can be found on the navigation bar above.
